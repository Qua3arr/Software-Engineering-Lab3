Для обработки ситуаций, когда запрос не обрабатывается ни одним из обработчиков, я добавлю в систему:
1. Класс DefaultHandler в конце цепочки, который будет обрабатывать все необработанные запросы (например, уведомлять администратора или сохранять в очередь на ручную проверку).
2. Изменю абстрактный класс Handler:
public abstract class Handler
{
    public abstract bool CanHandle(ReturnRequest request);
    public abstract void HandleRequest(ReturnRequest request);
}
Метод CanHandle позволит явно проверять возможность обработки.
3. В клиентском коде добавлю проверку после прохождения цепочки:
if (!request.IsProcessed)
{
    Console.WriteLine($"Запрос #{request.Id} требует ручной обработки");
    // Логика для необработанных запросов
}
