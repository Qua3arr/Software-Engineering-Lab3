Для хранения нескольких точек сохранения используется стек (Stack) в классе Caretaker. При каждом сохранении создается новый Memento и помещается в стек. При отмене извлекается последний сохраненный снимок.

Ограничения паттерна:
1. Потребление памяти - каждое сохранение хранит полную копию состояния
2. Сложность инкрементального сохранения - при больших объектах нужно сохранять только изменения
3. Необходимость сериализации - для сложных объектов требуется deep copy
4. Ограничение истории - без очистки старых снимков память растет бесконечно
5. Нарушение инкапсуляции - Memento может требовать доступа к приватным полям объекта
